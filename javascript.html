JavaScript Primitive Values

A primitive(primitive value, primitive data type) is data that is not an Object and has no methods. In JavaScript, there are six primitive data types: String, Number, Boolean, null, undefined, and Symbol (added in ECMAScript 2015)

All primitives are immutable(cannot be changed)

Most of the time a primitive value is represented directly at the lowest level of the language implementation.

Except for null and undefined, all primitive values have object equivalents that wrap around the primitive values:

  • String for the string primitive
  • Number for the number primitive
  • Boolean for the Boolean primitive
  • Symbol for the symbol primitive

JavaScript Types

JavaScript is a dynamic language. Another way of saying this is that It is loosely typed. This means that you do not have to define a variable's type when you declare it. It also means that you can reassign a variable to a different type.

There are seven data types under the current ECMAScript Standard:

  • Six that are primitives:
    • Boolean - true / false
    • Null - null only type
    • Undefined - unassigned variable
    • Number - double-precision 64-bit binary format
    • String - textual data
    • Symbol - unique - usually used as key of object property
  • and one compound data type:
    • Object
      • Collection of properties
      • Key/value pairs
      • Key can be a string or a symbol

JavaScript Type Conversions

The action of converting a primitive JavaScript value into a value of a different type is called a coercion or a conversion. Keep in mind that in JavaScript primitives are immutable, so JavaScript does not actually convert values. Instead, it returns a new value of the proper type.

Explicit Conversion

Strings to Numbers

The Number() function can be used to convert Strings that contain a Numeric value into a Number. The parseInt() and parseFloat() global functions turn Strings to Numbers, with each only handling numeric values in an integer or floating point format.

Number('1');          // 1
Number('abc');        // NaN
parseInt('123');      // 123
parseInt('123.12');   // 123 - will only return whole numbers
parseFloat('123.12'); // 123.12 - can handle floating point values

Numbers to Strings

You can use the String() function to convert numbers in to strings. You can also use call the toString() method on numbers.

You can also convert a Number to a String using the + operator to "add" it to a String. However, this code does not make its intent very clear and it is not recommended.

String(123);         // "123"
String(1.23);        // "1.23"
(123).toString();    // "123"
(123.12).toString(); // "123.12"
123 + '';            // "123"

Primitives to Boolean

If you have a String representation of a boolean you can compare it with the string 'true'.

var = 'true';
var = 'false';

a === 'true';  // true
b === 'true';  // false

You can also use the Boolean() function to explicitly convert a value to a Boolean. In JavaScript the following values convert to false:

  • null
  • NaN
  • 0
  • false
  • ''
  • undefined

All other values will covert to true.

Boolean(null);             // false
Boolean(NaN);              // false
Boolean('Hello, World!');  // true

Finally, The double ! operator can be used to coerce a truthy or falsy value to its boolean equivalent.

!!(null);   // false
!!('abc');  // true

Implicit Conversions

JavaScript has a number of implicit primitive type coercions. These implicit conversions are not always the most resonable and can be the source of bugs in your programs. It is best to not rely on the implicit conversions and use explicit conversions when needed. As a JavaScript developer you should know what the implicit conversions are so you have a better understanding of what the language is doing when it comes to troubleshooting and working with legacy code.

The (+) operator

When using the + operator with one operand (unary), it tries to convert values into Numbers.

+('123');  // 123
+('');     // 0
+(null);   // 0
+('a');    // NaN

The binary + operator (two oprands) means either addition for Numbers or concatenation for Strings. When + is used with mixed operands that include a string, JavaScript tries to convert the other operand to a string as well.

123 + '123'  // "123123"
'a' + null   // "anull"

The other arithmetic operators are only defined for Numbers, so JavaScript tries to coerce both operands into Numbers.

Relational Operators

The relational operators are defined for Numbers and Strings. Unless both operands are strings, JavaScript will try to do a number comparison.

Equality Operators

The non-strict equality operators have a complex set of rules to coerce types prior to comparing them. The strict operators never perform any sort of conversion.

Best Practices

Always use explicit type conversion and strict equality operators. Do not rely on JavaScript's implicit conversions.

Variable Scopes

A variable's scope is the portion of the program that can access that variable by name. In JavaScript, every function creates a new variable scope. When JavaScript tries to find a variable, it searches this hierarchy from the bottom to the top. It returns the first variable it finds with a matching name and stops searching.

With this searching order in mind, it is possiable for a variable in a lower scope to hide or shadow a variable with the same name in a higher scope.

Also note, during variable assignment, if the variable name does not exist, JavaScript creates the variable in the global scope.

Closures

A closure retains references to everything that is in scope when the closure is created and retains those references for as long as the closure exists.

Expressions

An expression is any valid code that resolves to a value. An expression can appear anywhere that JavaScript expects or allows a value

The most common expression types are:

  • Arithmetic - evaluates to a number
  • String - evaluates to a character string
  • Logical - evaluates to true or false
